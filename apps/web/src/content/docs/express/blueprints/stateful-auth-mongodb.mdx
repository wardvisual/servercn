# Stateful Authentication with MongoDB (Mongoose)

MongoDB implementation of stateful authentication using Mongoose ODM for session management.

## Installation Guide

You can add the stateful authentication blueprint to your project using the Servercn CLI:

<PackageManagerTabs command="npx servercn-cli add blueprint stateful-auth" />

## API Endpoints

All endpoints are the same for both MVC and Feature architectures.

## Key Files

### Controllers
- <Code>auth.controller.ts</Code> - Handles signup, signin, profile management, password operations, session management
- <Code>oauth.controller.ts</Code> - GitHub & Google OAuth callback handlers
- <Code>health.controller.ts</Code> - Basic and detailed health checks

### Services
- <Code>auth.service.ts</Code> - Core authentication business logic
- <Code>email.service.ts</Code> - Email delivery using Nodemailer
- <Code>otp.service.ts</Code> - OTP generation, hashing, and verification
- <Code>upload.service.ts</Code> - Cloudinary file upload integration

### Models
- <Code>user.model.ts</Code> - User schema with hooks and instance methods
- <Code>session.model.ts</Code> - Session tracking schema
- <Code>otp.model.ts</Code> - One-time password schema

### Routes
- <Code>auth.routes.ts</Code> - All authentication endpoints
- <Code>oauth.routes.ts</Code> - OAuth flow routes
- <Code>health.routes.ts</Code> - Health check endpoints
